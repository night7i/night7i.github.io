什么是information_schema？

在MySQL5.0版本以上，MySQL会自带一个名为information_schema的虚拟数据库，它提供了数据库元数据的方式，即数据的数据；如数据库名、数据库的表、表栏的数据类型与访问权限等。

在informaiton_schema中，有数个只读表。它们实际上是视图，而不是基本表，因此，你将无法看到与之相关的

查看数据库名

在调用information_schema数据库后

使用SHOW TABLES;查看所有表

schemata表内SCHEMA_NAME字段内储存了所有的数据库名 输入语句查看所有库

mysql> select group_concat(SCHEMA_NAME) from information_schema.SCHEMATA;

查看表名

TABLES表内TABLE_NAME字段储存了所有表的名称 输入语句查询某一数据库下的所有表

mysql> select TABLE_NAME from information_schema.TABLES where TABLE_SCHEMA='库名';

查看表下所有的列

COLUMNS表内COLUMN_NAME字段储存了所有的字段名称 输入语句查询某表下所有字段

mysql> select COLUMN_NAME from information_schema.COLUMNS where TABLE_SCHEMA=database() and TABLE_NAME='表名'

利用联合查询以及information_schema即可获取到数据库内的信息，如果页面有回显值，在页面的sql语句后跟联合查询即可直接查询数据，如果没有回显值，或者属于报错型注入则需要通过盲猜获取数据
